### **Aim**
> To monitor disk usage and manage partitions and file systems


### **Theory**
- **`df` (Disk Free)**
  - Displays the amount of available and used disk space on file systems.
  - *Syntax:* `df [options] [file]` (e.g., `-h` for human-readable format).

- **`du` (Disk Usage)**
  - Estimates and displays the file space usage for a specific directory or file.
  - *Syntax:* `du [options] [directory]` (e.g., `-sh` for a human-readable summary).

- **`fdisk`**
  - A command-line utility used for disk partition manipulation (creating, deleting, formatting partitions).
  - *Syntax:* `sudo fdisk [device_name]` (e.g., /dev/sda)

- **`mkfs` (Make File System)**
  - Used to build a Linux file system (like ext4) on a formatted partition.
  - *Syntax:* `sudo mkfs -t [fstype] [device_name]`

- **`mount` / `umount`**
  - `mount` attaches a storage device to a specific directory tree (mount point). `umount` safely detaches it.
  - *Syntax:* `sudo mount [device] [directory]` / `sudo umount [directory]`


### **Commands**
```bash

$ df -h
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   20G   28G  42% /
tmpfs           3.9G     0  3.9G   0% /dev/shm

$ du -sh /home/student/Documents
1.2G    /home/student/Documents

$ sudo fdisk -l
(Lists all available disk partitions and their sizes)

$ sudo fdisk /dev/sdb
(Opens the fdisk interactive menu. Type 'n' for new partition, 'w' to write changes)

$ sudo mkfs -t ext4 /dev/sdb1
(Formats the newly created partition to the ext4 file system)

$ sudo mkdir /mnt/usb_drive
$ sudo mount /dev/sdb1 /mnt/usb_drive

$ df -h | grep sdb1
/dev/sdb1        10G   24K  9.9G   1% /mnt/usb_drive

$ sudo umount /mnt/usb_drive

```


### **Conclusion**
> The utilities for storage management were successfully executed. System storage was monitored using df and du, and the lifecycle of a disk partition—from creation (`fdisk`) and formatting (`mkfs`) to mounting and unmounting (`mount`, `umount`)—was practically understood
